# RISK-01 / RISK-02 - Contract Specs, Costs, Sizing, Risk Gate

Status: implemented

## Scope

This risk layer is simulation-first. It defines deterministic contract math, venue-cost assumptions, a round-trip cost model, a phase-one composed position sizer, and a narrow risk-gate helper for ORCH-02.

It does not implement live order routing, broker connectivity, account mutation, sockets, or a runner.

## Contract Specs

`apps/strategy_runtime/src/risk/contracts.ts` defines CME equity-index futures specs used by the simulator and research/replay tools:

- `MNQ`
- `NQ`
- `MES`
- `ES`

V1 runtime config remains MNQ-centered. Additional roots are present only so cost and continuity logic can be explicit when research tickets need them. Every spec has `live_order_routing_allowed: false`.

## Venue Costs

`config/venue-costs.json` contains V1 simulation assumptions for per-side commission and exchange fees. The loader in `venue-cost-config.ts` validates every supported contract root and returns an immutable table shape with source lineage fields.

These values are not credentials and are not broker authorization. They must be reviewed before any future live-routing scope.

## Cost Model

`costs.ts` computes:

- commission and exchange fees
- adverse entry and exit slippage
- planned gross PnL
- actual gross PnL
- actual net PnL after commission and exchange fees
- optional gross/net R multiples

All timestamps remain caller-owned; this module uses no wall-clock time.

## Composed Sizer

`composed-sizer.ts` implements the phase-one formula:

```text
q_final = floor(min(q_risk, q_liq, q_softcap, q_hardcap, q_net_cap))
```

The components are returned for journaling and operator explainability. Liquidity hysteresis is deterministic and size-based; callers provide `now_ms` explicitly when they want cooldown behavior. There is no `Date.now()`.

Phase one intentionally has no Kelly term. `q_kelly` is returned as `null` for stable schema shape until calibrated paper-trade evidence exists.

## Risk Manager

`risk-manager.ts` adapts active `Candidate` contracts into:

- `SizingDecision`
- `RiskGateDecision`

The risk gate checks:

- candidate is still `proposed`
- minimum reward-risk threshold
- non-zero sizing
- max net position per symbol
- daily realized-loss limit

RISK-03 will expand this with daily open-trade count and account/session circuit-breaker transitions.
